Structurely · Pricing Model Flow

At signup: implementation fee, subscription, and a prepaid token amount for the cycle — subscription and token limits share one rolling cycle from signup.

If token usage hits the limit before the cycle ends: pause until the next cycle or agree to daily overage (calculated each day, charged in Stripe the following day). Unpaid overage pauses the account until it is settled.

Step 1 · Sales
Scope the Plan Before Signing
Sales works with the prospect to choose the right subscription tier and a realistic monthly token amount (and budget).
Step 2 · Signup & Payment
Implementation + Subscription + Prepaid Token Amount
At signup the client pays the implementation fee, subscription, and the agreed prepaid token amount for the cycle. Subscription renewals and token limits reset on the same rolling cycle (anchored to signup).
Step 3 · Usage Monitoring
Watch Token Usage Against Token Limits
Monitor token usage so you can see who is approaching their token limit and ensure proactive outreach before that limit is reached.
~90% of token usage reached
Step 4 · Trigger
Sales Reaches Out to Client
Once a client reaches ~90% of token usage: Proactive touchpoint — present options, understand client intent for the remainder of the billing cycle.
Option A
Stay Within Token Limits
Pause Until Next Cycle
Once token usage reaches the token limit, Outbound is paused for the rest of the cycle. Inbound stays operational with today's product limits. No further outbound use until the next cycle begins.
Option B
Daily Overage
Client Agrees to Continue Past the Token Limit
Overage is calculated per day; each day's overage is charged in Stripe the following day. Miss an overage payment → account pauses until that balance is paid.
⚠ Non-Payment (Option B)
Account Paused Until Overage Is Paid
Unpaid overage amounts pause the account until settled. Outbound is shut off via feature flag; Inbound stays operational with today's product limits.
Step 6 · Cycle Repeats
New Billing Cycle Begins — Reset & Review
If a client is consistently at or above their token limit, sales should work with them to raise their standard subscription and monthly token amount.

Note: Clients cannot lower their token limits themselves; changes go through sales/CS.
